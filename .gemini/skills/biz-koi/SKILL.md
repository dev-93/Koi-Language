# 💖 biz-koi: 일본어 대화 가이드 및 UI 전문가

이 스킬은 단순 학습을 넘어 관계의 진전을 만드는 일본어 대화 시스템과 이를 구현하기 위한 UI 가이드라인을 지원합니다.

## 👔 PM 전략 및 기획 지침

1. **효율적 중복 체크**: 새로운 상황 설계 전, 노션 DB에서 전체 데이터가 아닌 **최근 20개의 테마 목록(Title)**만 조회하여 토큰을 아끼십시오. 기존에 없던 '신선한 관계의 긴장감'을 찾는 데 집중하십시오.
2. **마케팅 기획 우선**: 상황을 만들며 동시에 **"이 상황을 어디에(숏폼/Threads/카페), 어떤 문구로 홍보할 것인가"**를 확정하십시오. 홍보할 가치가 없는 평범한 상황은 기획하지 마십시오.
3. **수익 근거 포함**: 보고 시 해당 시나리오가 왜 유료 전환(Premium)에 유리한지 마케팅 관점의 근거를 짧고 강력하게 제시하십시오.

## 🎯 핵심 목표

- Gemini 2.5를 활용한 상황별 맞춤형 감정 온도 분석.
- Freemium 모델 구축 및 리드 마그넷 PDF 배포.
- 프로젝트 전체의 일관된 UI/UX 유지 (Peach/Lavender 테마).

## 📋 워크플로우

1. **뉘앙스 분석**: 사용자가 입력한 일본어 문구의 상세 감정 및 뉘앙스 분석.
2. **마케팅 자동화**: 숏폼(TikTok/Shorts) 문구 추출 및 노출 최적화.
3. **유료화 전환**: 프리미엄 기능(감정 Tip 등)의 마스킹 처리 및 클릭률 측정.
4. **UI 구현**: 새로운 화면이나 컴포넌트 생성 시 프로젝트의 디자인 시스템 준수.
5. **즐겨찾기 시스템**: 학습 중 마음에 드는 표현을 하트로 저장, 홈 화면 '내 표현' 탭에서 재열람 가능. Zustand persist로 로컬 저장.

## ✍️ 커밋 규칙

- 커밋 메시지는 **항상 한글**로 작성합니다.
- 형식: `타입: 설명` (예: `기능: 데이팅 팁 토글 추가`, `수정: 하단 네비 높이 축소`, `스타일: 메인 색상 변경`)

## 📱 모바일 UI 원칙

- **스크롤 최소화**: 모바일(480px 이하)에서 핵심 화면은 스크롤 없이 한 화면에 표시되어야 합니다.
- **반응형 분리**: PC와 모바일 스타일은 반드시 CSS 미디어 쿼리(`@media (max-width: 480px)`)로 분리합니다. 인라인 스타일로 반응형 처리 금지.
- **고정 영역 최소화**: `position: fixed` 하단 영역(네비, 버튼 등)은 최대한 얇게 유지합니다.
- **overflow 제어**: 모바일 레이아웃 최상위 컨테이너에 `overflow: hidden`과 `height: 100dvh`를 적용합니다.
- **폰트 크기**: 모바일에서는 PC 대비 2~4px 작게 설정합니다.

## 💾 상태 관리 (Zustand Store)

- **persist 미들웨어**: `koi-language-storage` 키로 localStorage에 자동 저장.
- **favorites**: 마음에 드는 표현 배열. `toggleFavorite(expr)` / `isFavorite(exprId)` 메서드 사용.
- **dailyProgress**: 일일 학습 진행도 (날짜 변경 시 자동 리셋).
- **currentTab**: 홈 화면 탭 상태 (`today` | `archive` | `favorites` | `settings`).
- **recentKeywords**: 중복 방지용 최근 키워드 (최대 8개).

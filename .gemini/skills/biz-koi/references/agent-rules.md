# 🤖 Koi-Language AI 에이전트 지침서 (v1.0)

이 문서는 **Koi-Language** 프로젝트를 수행하는 모든 AI 에이전트가 반드시 숙지하고 준수해야 할 핵심 정의와 규칙을 담고 있습니다.

## 🎯 1. 프로젝트 정의 및 목표
- **서비스명:** Koi-Language (恋의 런지)
- **슬로건:** "단순 공부가 아닌, 관계의 진전을 만드는 실전 일본어"
- **핵심 가치:** 
  - 일본인 연인/친구와 실제 데이트 상황에서 바로 쓸 수 있는 생생함.
  - 단순 번역을 넘어선 '문화적 맥락'과 '감정의 온도(Tip)' 전달.
  - 전 세계 사용자를 위한 로마자(Romaji) 발음 데이터 제공.

## ⚠️ 2. 핵심 운영 규칙 (MUST FOLLOW)

### 2.1 일일 상황 제한 (Strictly One Situation Per Day)
- **규칙:** Notion 데이터베이스의 `Date` 필드 기준으로, **하루에 단 한 개의 상황(Situation)**만 존재해야 합니다.
- **주의:** 수동 실행(`scripts/run-cron-now.js`) 시 중복 실행되지 않도록 주의하며, 기존 데이터가 있는지 반드시 먼저 확인합니다.

### 2.2 날짜 설정 원칙
- **기준 시간:** 한국 시각(KST, UTC+9) 기준.
- **날짜 값:** 크론 실행 시점의 **'현재 날짜(오늘)'**를 기본값으로 사용합니다.

### 2.3 데이터 구조 및 품질
- **상황 (Situation):** 상황 제목(`Title_KR/JP`)과 상세 설명(`Desc_KR/JP`)이 풍부해야 함.
- **표현 (Expression):**
  - 타겟팅(`Target`): 가급적 `INTEGRATED`를 사용하여 양방향 사용자 모두에게 노출.
  - 발음(`Reading`): 전용 도구(Romaji)를 활용한 정확한 로마자 표기.
  - 팁(`Tip`): `{ "kr": "...", "jp": "..." }` JSON 형식을 준수하여 문화적/심리적 조언 포함.
  - 단어(`Words`): 해당 표현의 핵심 어휘 최대 3개까지 JSON 배열 형태로 포함.

## 🛠️ 3. 기술적 제약 사항
- **환경 변수:** `VITE_` 접두사가 제거된 최신 변수명(`GEMINI_API_KEY`, `NOTION_TOKEN` 등)을 사용.
- **에러 보고:** 작업 완료 또는 실패 시 텔레그램 비즈니스 리포트 스크립트(`scripts/report-business.js`)를 통해 현황을 보고.

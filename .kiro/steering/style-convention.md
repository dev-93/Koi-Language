# 스타일 컨벤션

## 원칙

- 컴포넌트 스타일은 `<style jsx>`로 관리한다.
- Tailwind 유틸리티 클래스(`w-px`, `h-6`, `mt-8`, `gap-3` 등)를 직접 사용하지 않는다.
- 인라인 `style={{ }}` 어트리뷰트를 사용하지 않는다.
- `globals.css`는 리셋, 폰트, CSS 변수 등 전역 설정만 담는다.

## className 네이밍

- 의미 있는 이름을 사용한다 (예: `nav-divider`, `card-title`, `learn-button`)
- `w-px h-6 bg-gray-100` 같은 약어 조합을 쓰지 않는다.

## 구조

- 각 컴포넌트(`.jsx`) 파일 하단에 `<style jsx>` 블록을 둔다.
- 공통으로 재사용되는 스타일만 `globals.css`에 정의한다.

## 기존 코드 수정 시

- 새로 작성하거나 수정하는 부분은 이 컨벤션을 따른다.
- 기존 Tailwind/인라인 스타일을 한꺼번에 마이그레이션하지 않는다. 수정이 필요한 부분만 점진적으로 전환한다.

# 커밋 컨벤션

## 원칙

- 커밋은 기능/수정 단위로 분리한다. 하나의 커밋에 여러 목적의 변경을 섞지 않는다.
- 커밋 메시지는 한글로 작성한다.

## 분리 기준

- 버그 수정과 기능 추가는 별도 커밋
- 리팩토링과 기능 변경은 별도 커밋
- steering/skills 등 설정 파일 변경은 별도 커밋
- 스타일(CSS/UI) 변경은 별도 커밋

## 메시지 형식

- `feat: 설명` — 새 기능
- `fix: 설명` — 버그 수정
- `refactor: 설명` — 리팩토링
- `style: 설명` — UI/스타일 변경
- `chore: 설명` — 설정, 스크립트, 문서 등
